$.engineer.define( object_type, definition )

Let's have a look at the definition. Each definition can have 3 parts: structure, behavior and defaults. But none of them are required.

Structure: function([options]) { return object; }

Structure is defined as a function that can optionally take on parameter for options. What ever is returned from the structure function will be the objects DOM element(s). In the example above the structure function returns a div:

Behavior: function([options]) { return methods; }

Behavior is also defined as a function. The context of the behavior function is the jQuery object returned by the structure function. No need to wrap this in $(). Since this is already a jQuery object it's easy to bind events to your element. Notice that the elements created above all have a click function bound to them. Scroll up and give it a try.

The behavior function can return methods that can be invoke later. More on this in the Object Methods section.

Defaults:

You may have noticed both the structure and behavior functions both take a single parameter: options. Defaults set up the default options that are passed into structure and behavior. You can override these by passing new options when you create your element. These new options will be merged into your defaults.

$.engineer.build(object_type, [options])

If you only want the structure of a engineer objects you can use the build() method. Below I've only created the structure of the button not the behavior.

$().makeInto(object_type);

If you'd like to take some existing DOM elements and make them into jQuery engineer objects you can use the makeInto() method.

You don't need to pass any options into makeInto. For each option defined in defaults, makeInto will take the xhtml attributes of the elements as it's options.
