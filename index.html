jQuery.ENGINEER

provides a clean and objective way to control the structure (HTML/CSS) and behavior (JS) of DOM elements, with the help of jQuery.

Why?

Programming for the web can be a confusing tangle of old and new programming languages and techniques. Building rich applications for web browsers requires HTML, CSS, and JavaScript to all work together. In most projects their responsibilities overlap heavily. This juggling act can get hard quickly, especially when DOM objects with stateful JS behavior have to talk to each other or be duplicated.

JQuery.engineer aims to package complex interactive behaviors into modular, reusable and clear building blocks. With engineer, the question of responsibility becomes much easier, and code you already wrote can be reused, simplified and refactored with ease.

$.engineer.define( object_type, definition )

Define an object structure and behavior set to be built by engineer.

Let's have a look at the definition. If you're used to OOP, it's easy to think of these like Class definitions, or like the Factory pattern. Each definition can have 3 parts: structure, behavior and defaults. None of them are required (though it would certainly be silly to not use at least one).

Structure: function([options]) { return object; }

Structure should be a function that can optionally take one parameter for options. Structure must return the object(s) you want to use as DOM element(s). In the example below the structure function returns a div:

Behavior: function([options]) { return methods; }

Behavior should be a function that can optionally take one parameter for options. The context of the behavior function is the jQuery object returned by the structure function. There's no need to wrap this in $(). Since this is already a jQuery object it's easy to bind events to your element. If you like, you can always un-JQuery and get at the original JS object with this[0].

The behavior function can return methods to be called later. More on this in the Object Methods section.

Defaults:

You may have noticed both the structure() and behavior() functions both take a single parameter: options. Defaults define the default options that are passed into structure() and behavior(). You can override these by passing new options when you create your element. These new options will be merged into your defaults.

$().makeInto(object_type);

Take some existing DOM elements and make them into jQuery engineer objects (object_type) with makeInto().

You don't need to pass any options into makeInto(). For each option defined in defaults, makeInto will take the xhtml attributes of the elements as its options.
