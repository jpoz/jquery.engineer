Let's have a look at the definition. Each definition can have 3 parts: structure, behavior and defaults. But none of them are required.

Defining structure:

Structure is defined as a function that can optionally take on parameter for options. What ever is returned from the structure function will be the objects DOM element(s). In the example above the structure function returns a div:

Defining behavior:

Behavior is also defined as a function. The context of the behavior function is the jQuery object returned by the structure function. No need to wrap this in $(). Since this is already a jQuery object it's easy to bind events to your element. Notice that the elements created above all have a click function bound to them. Scroll up and give it a try.

Setting options:

You may have noticed both the structure and behavior functions both take a single parameter: options. The defaults key sets up your default options. You can over ride these by passing new options when you create your element. These new options will be merged into your defaults.

send()

To invoke a public method you use send() method on any jQuery object(s). If the object does not have a public method it will not error so you can pass groups of objects public methods without worry. Here's the click function of the button above.

Send can also pass arguments to public methods. If a public method takes arguments add them to the arguments you give send. For Rubyists this is a lot like the ruby send method.
